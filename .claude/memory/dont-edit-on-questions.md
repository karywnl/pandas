---
name: dont-edit-on-questions
description: When the user asks a question or says something confuses them, diagnose and discuss first; do not jump straight to editing files
metadata:
  type: feedback
---

When the user raises a point of confusion or asks a question about page content (e.g. "this quick check confuses me because..."), do NOT immediately start editing the file. Diagnose, explain, and confirm whether a change is actually wanted first.

**Why:** The user may be working through their own understanding, or the content may be correct as written. In one case the user's confusion was self-resolved ("my bad, the df was mentioned") and an unprompted edit would have broken correct content. The user explicitly objected to me starting to change things before they asked.

**How to apply:** Treat questions as questions, not as edit requests. Only edit once the user has said they want a change.
